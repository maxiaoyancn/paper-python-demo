# 数据处理逻辑

> 描述每一行 spec 从输入 Excel 到回写输出之间，**数据本身**经历了哪些数学/统计变换。
> 不涉及脚本结构、CLI、文件 IO；只讲数据怎么被算出来。

---

## 0. 数据契约

每一行 spec 给定的输入是 4 类标量：

| 名称 | 类型 | 范围 | 含义 |
|---|---|---|---|
| `K` | int | 2..26 | 组数（由表头位置推断，不是行级） |
| `N_i` (i=1..K) | int | ≥2 | 第 i 组样本量 |
| `μ_i` (i=1..K) | float | 任意 | 第 i 组目标均值 |
| `σ_i` (i=1..K) | float | >0 | 第 i 组目标标准差 |
| `decimals` | int | ≥0 | 输出原始数据的小数位数 |

输出（写回到 Excel 同一行）的 5 大块：

1. K 组实际原始数据（每组 N_i 个数）
2. K 组实际回算的 `(μ_actual, σ_actual)`（共 2K 个数）
3. 4 个组级 p 值与 2 个 Y/N 结论：`Levene p`、`是否方差齐`、`Shapiro-Wilk min p`、`是否正态`、`整体 p`
4. C(K,2) 个两两 raw p
5. C(K,2) 个两两 Q-value（多重比较校正）

整个数据流水线对每一行**独立**执行，行与行之间无耦合。

---

## 1. 生成单组数据：精确还原 μ 与 σ

输入：`μ_target, σ_target, N, rng, decimals`。
输出：长度为 N 的数组 `x`，使 `mean(x) ≈ μ_target`、`std(x, ddof=1) ≈ σ_target`（精度受 `decimals` 量化限制）。

### 1.1 标准正态采样

```
z = rng.standard_normal(N)         # 来自 N(0,1) 的 N 个 iid 样本
```

样本均值 `z̄` 与样本标准差 `s_z = std(z, ddof=1)` 通常**不会**精确等于 0 与 1（有限样本误差）。

### 1.2 强制中心化与单位化（关键步骤）

```
z' = (z - z̄) / s_z
```

数学事实：经过 `(x - x̄) / s_x` 后，**任意有限样本** `z'` 都满足

```
mean(z') = 0
std(z', ddof=1) = 1
```

无误差，是恒等式。

### 1.3 线性变换到目标位置与刻度

```
x_raw = μ_target + σ_target × z'
```

线性变换保 mean 与 SD 的伸缩律：

```
mean(x_raw) = μ_target + σ_target × 0 = μ_target
std(x_raw, ddof=1) = |σ_target| × 1 = σ_target
```

**此时 `x_raw` 的均值与标准差严格等于目标值**——还没有四舍五入。

### 1.4 量化（round）

```
x = round(x_raw, decimals)
```

四舍五入会引入微小扰动，导致：

```
mean(x) ≈ μ_target   （偏差通常 < 10^-decimals 级别）
std(x, ddof=1) ≈ σ_target （偏差不可忽略，尤其 decimals=0 时）
```

均值的扰动对论文表格几乎不可见；SD 的扰动可能超出可接受范围，需要下一步兜底。

---

## 2. SD ±10% 容差重试

量化后实际 SD 记为 `σ_actual = std(x, ddof=1)`，相对误差：

```
rel_err = |σ_actual - σ_target| / σ_target
```

判定：
- `rel_err ≤ 0.10`（即 ±10%）→ 接受这一组数据
- `rel_err > 0.10` → 用主 rng 派生**新的子种子**，回到第 1 步重新采样
- 累计重试 5 次仍失败 → 抛 `RuntimeError(metric, target_sd, last_actual_sd, retries=5)`，本行作废、整个流水线终止

容差重试是**单组级**：每组各自独立判 ±10%；某组失败不会影响其它组的已通过结果（实际上整个流水线都崩了，所以无所谓"已通过结果"）。

容易触发重试的场景：
- `decimals = 0` 且 `σ_target` 很小（如 ≤1.0）—— 量化粒度比目标 SD 还粗
- `N` 很小（如 N=3）—— 单点扰动占比大

## 3. 组级统计量

K 组数据生成完毕后，对全部 K 个数组 `[g_1, g_2, ..., g_K]` 算下面 4 类统计。

### 3.1 每组的实际 mean / SD（回算）

```
μ_actual_i = mean(g_i)
σ_actual_i = std(g_i, ddof=1)
```

写入对应的"<letter> 组均值"与"<letter> 组SD值"，4 位小数（`round(v, 4)`）。

### 3.2 Levene 方差齐性检验

调 `scipy.stats.levene(*groups, center='median')`——**Brown-Forsythe 变种**（用中位数代替均值做绝对离差，比经典 Levene 更鲁棒，适合非正态分布）。

返回 `levene_p`：

| levene_p | 解读 | 标志 `equal_var` | "是否方差齐"列 |
|---|---|---|---|
| `> 0.05` | 不能拒绝"方差齐"原假设 | True | `Y` |
| `≤ 0.05` | 拒绝原假设，方差不齐 | False | `N` |
| `NaN`（罕见，例如某组方差 0） | 视作不齐 | False | `N` |

### 3.3 Shapiro-Wilk 正态性检验

对**每一组**单独跑 `scipy.stats.shapiro(g_i)`，得到 K 个 p 值，取最小：

```
sw_min_p = min(p_1, p_2, ..., p_K)
```

特殊处理：
- `N_i < 3` → SW 不可计算，该组 p 值视作 `0`（不正态）+ warning
- SW 抛异常或返回 NaN → 同样视作 `0` + warning

判定与"是否正态"列：

| sw_min_p | 解读 | 标志 `all_normal` | "是否正态"列 |
|---|---|---|---|
| `> 0.05` | 全部 K 组都通过正态性 | True | `Y` |
| `≤ 0.05` | 至少 1 组未通过 | False | `N` |

注意：**取最小**意味着只要有一组不正态，整行就视作"非全正态"，这会影响后续两两对比的方法选择。

### 3.4 整体差异 p（K 组之间）

二选一，由 `equal_var` 决定：

| 条件 | 方法 | 函数 | 含义 |
|---|---|---|---|
| `equal_var = True`（方差齐） | 单因素方差分析 | `scipy.stats.f_oneway(*groups)` | 参数法，假设方差齐 + 近似正态 |
| `equal_var = False`（方差不齐） | Kruskal-Wallis 检验 | `scipy.stats.kruskal(*groups)` | 非参数法，不要求方差齐与正态 |

**注意**：整体差异分支**只看 Levene**，不看 SW。即使 SW 全过、但 Levene 不齐，仍走 KW；反之 SW 不过、但 Levene 齐，仍走 ANOVA。

p 值取 4 位小数后写入"整体 p（ANOVA/KW）"列。

### 3.5 两两对比：Tukey HSD vs Welch+Bonferroni

二选一，需要**同时满足两个条件**才走 Tukey 分支：

```
if equal_var AND all_normal:
    # 分支 A：Tukey HSD
else:
    # 分支 B：Welch+Bonferroni
```

#### 分支 A：Tukey HSD

```
matrix = scipy.stats.tukey_hsd(*groups).pvalue   # 形状 (K, K)，对称矩阵
raw[(i,j)] = matrix[i][j]                         # 取上三角，按 (1,2)(1,3)(2,3)(1,4)... 顺序
Q = None  对所有 pair
```

Tukey HSD 内置多重比较校正，所以 raw p 已经是校正后的家族错误率控制 p 值；**Q-value 列留空**（写 None）。

#### 分支 B：Welch t + Bonferroni

对每对 `(i, j)`：

```
raw[(i,j)] = scipy.stats.ttest_ind(g_i, g_j, equal_var=False).pvalue
n_pairs = C(K,2)
Q[(i,j)] = min(1.0, raw[(i,j)] × n_pairs)
```

Welch t 不要求方差齐；Bonferroni 把 raw p 简单乘以对数（封顶到 1.0）。

#### 配对顺序（K=4 例）

`itertools.combinations(range(K), 2)` 产生：

```
(0,1) (0,2) (0,3) (1,2) (1,3) (2,3)
 A-B   A-C   A-D   B-C   B-D   C-D
```

raw p 列与 Q-value 列**同序**。

### 3.6 决策真值表

| Levene 齐 | SW 全正态 | 整体 p 用 | 两两对比用 | 是否方差齐 | 是否正态 |
|:---:|:---:|---|---|:---:|:---:|
| Y | Y | ANOVA | Tukey HSD | Y | Y |
| Y | N | ANOVA | Welch + Bonferroni | Y | N |
| N | Y | KW | Welch + Bonferroni | N | Y |
| N | N | KW | Welch + Bonferroni | N | N |

只有第 1 行（齐 + 全正态）才能走 Tukey；其他三种情况都走 Welch+Bonferroni。

## 4. 数值精度与 round 策略

| 数据 | 精度 | round 时机 |
|---|---|---|
| 原始数据（数据区） | 行级 `decimals` | 在 `np.round(x, decimals)` 时 |
| 每组实际 μ / σ | 固定 4 位小数 | 写回 cell 前 `round(v, 4)` |
| Levene p / SW min p / 整体 p / raw p / Q-value | 固定 4 位小数 | 同上 |
| `Y/N` 结论列 | 字符 | 直接写 `"Y"` 或 `"N"` |
| `NaN` 数值（任意来源） | — | 写为 `None`（空 cell） |

## 5. 可复现性的数学保证

主 rng 由 `--seed` 决定（不传则 `secrets.randbits(32)` 随机）。

每一组数据生成前，从主 rng 派生**一个 32-bit 子种子**：

```
sub_seed = int(rng.integers(0, 2**32 - 1))
sub_rng = np.random.default_rng(sub_seed)
```

派生顺序是**严格按行号 + 组序号**：行 2 的 G1、G2、…、G_K，行 3 的 G1、G2、…，依此类推。

性质：
- 同 `--seed` + 同输入 → 主 rng 状态一致 → 子种子序列一致 → 每组 z 序列一致 → 量化后 x 序列一致 → 所有统计 p 值一致 → 写回的每个 cell 完全一致
- SD 容差重试也走同一个主 rng，每次重试就是再 `rng.integers(...)` 派生一个新子种子；所以"重试路径"也是可复现的——同 seed 下，第几次重试成功、用的是哪个子种子，都确定

## 6. 一行的端到端示例（K=2，N=10/12，σ_2 较接近 σ_1）

输入（行）：

```
metric = "体重"
G1: N=10, μ=120.6263, σ=10.3698
G2: N=12, μ=110.36,    σ=9.8635
decimals = 4
```

执行：

```
rng = default_rng(seed=42)

# G1
sub_seed_1 = int(rng.integers(...))
z_1   = standard_normal(10)
z_1'  = (z_1 - z̄_1) / s_z1                # 强制 mean=0, SD=1
x_1   = 120.6263 + 10.3698 × z_1'
x_1   = round(x_1, 4)                       # decimals=4
σ_actual_1 = std(x_1, ddof=1)
rel_err = |σ_actual_1 - 10.3698| / 10.3698
# 假设 rel_err ≤ 0.10 → 接受

# G2
sub_seed_2 = int(rng.integers(...))
... 同上，得 x_2

# 统计
levene_p, equal_var = levene(x_1, x_2, center='median')   # 例如 0.9294, True
sw_min_p, all_normal = min(shapiro(x_1).p, shapiro(x_2).p), all_pass   # 例如 0.43, True
overall_p = f_oneway(x_1, x_2).pvalue                       # 走 ANOVA：例如 0.0276
raw, q, _ = tukey_hsd(x_1, x_2)                             # 走 Tukey：raw=0.0276 (K=2 时与 ANOVA 等价), q=None

# 写回
[A1..A10] = x_1
[B1..B12] = x_2
[A 组均值] = round(mean(x_1), 4)
[A 组SD值] = round(std(x_1, ddof=1), 4)
[B 组均值] = round(mean(x_2), 4)
[B 组SD值] = round(std(x_2, ddof=1), 4)
[Levene p] = 0.9294
[是否方差齐] = "Y"
[Shapiro-Wilk min p] = 0.43
[是否正态] = "Y"
[整体 p（ANOVA/KW）] = 0.0276
[A-B raw p] = 0.0276
[A-B Q-value] = None    # Tukey 分支
```

## 7. 极端情况下的数据处理路径

| 场景 | 数据处理结果 |
|---|---|
| 某组 σ_target 太小 + decimals=0 | round 完 SD 与 σ_target 相对误差 > 10% → 重试；5 次仍失败 → 整个脚本 RuntimeError 崩溃，**该行的数据区与统计区保持原状不变** |
| K=2 + N1=N2=2 | 数据生成正常；SW 因 N<3 视作 p=0、`all_normal=False` + warning；整体 p 走 ANOVA（如果 Levene 齐）；两两走 Welch+Bonferroni（因为 all_normal=False）；K=2 时 Welch+Bonferroni 的 Q = raw × 1 = raw |
| 某组实际全是同一个数（理论上 std=0） | 不会触发——线性变换前 z' 已经强制 SD=1，σ_target>0，`x = μ + σ × z'` 不可能让所有元素相等。但若 decimals 很粗 + N 很小，round 后所有数恰好相同，会让 Levene 行为退化（p 可能 NaN），按规则视作不齐 → "是否方差齐"= N |
| K 组目标 σ 完全相等 | Levene 几乎一定 p > 0.05（齐）；走 ANOVA；若 SW 也都过 → Tukey |
| K 组目标 σ 差异大（max/min ≥ 3） | Levene 大概率 p ≤ 0.05（不齐）；走 KW + Welch+Bonferroni |
| 行内 N≤1 / SD≤0 / decimals 缺失 | `read_specs` 阶段 warning + 跳过该行；数据区与统计区**保持原状**，不写入；后续行继续处理 |

## 8. 数据流的不变量（恒等式）

不论 seed、N、K 如何，下面的等式总成立：

1. **量化前**：`mean(x_raw) = μ_target` 且 `std(x_raw, ddof=1) = σ_target`（严格相等）
2. **量化后**：`mean(x) = μ_target ± O(10^-decimals)`，`std(x, ddof=1) = σ_target × (1 ± rel_err)` 其中 `rel_err ≤ 0.10`（接受时）
3. **回算列与原始数据一致**：写到"<letter> 组均值"和"<letter> 组SD值"的就是 `mean(x)` 与 `std(x, ddof=1)`，与数据区里 N_i 个 cell 一一对应
4. **Y/N 结论与 p 值一致**：`是否方差齐 == "Y"` ⟺ `Levene p > 0.05`；`是否正态 == "Y"` ⟺ `SW min p > 0.05`
5. **整体 p 分支**：`equal_var = True` ⟺ 走 ANOVA；否则走 KW
6. **两两分支**：走 Tukey ⟺ `equal_var AND all_normal`；否则走 Welch+Bonferroni
7. **Tukey 分支 Q-value 全为空**；**Welch+Bonferroni 分支 Q-value 全有值且 ≤ 1.0**
8. **可复现性**：同 seed + 同输入 ⟹ 每个 cell 完全相等

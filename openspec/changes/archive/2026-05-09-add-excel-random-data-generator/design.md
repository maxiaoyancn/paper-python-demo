## Context

- 项目目前是一个空的 Python 脚本工作区（参见根 `CLAUDE.md` §七）。仓库根目录有一份 Excel：`20260509-随机数生成.xlsx`，由用户从论文/实验表格里整理而来。
- Sheet 结构（第 1 行是表头，第 2 行起每行是一个指标）：
  - **A 列** 指标名（如「体重」「身高」「血糖」）。
  - **B 列** 第一组样本量 N，**C 列** 第一组目标均值，**D 列** 第一组目标 SD。
  - **E 列** 第二组样本量 M，**F 列** 第二组目标均值，**G 列** 第二组目标 SD。
  - **H 列** 是用户写的备注（要求文本），不应改动。
  - **I 列** 起是第一组生成数据，长度 = N，**第 1 行**只在最后一个数据列写 `<count>（N）`（中文括号），其它列写 1..N-1。
  - 第一组之后**空一列**（即 `I 列号 + N` 那一列），第二组从下一列开始（在样本数据中是 T 列），长度 = M，**第 1 行**最后一格写 `<count>（M）`。
  - 第二组之后**再空一列**，紧跟 5 个统计列：第一组均值、第一组 SD、第二组均值、第二组 SD、p 值（在样本数据中是 AG-AK）。表头在第 1 行已经存在，无需重写。
- 输入示例（B=10、E=12）：第一组占 I-R（10 列），空 S，第二组占 T-AE（12 列），空 AF，统计 AG-AK。脚本必须能在 N、M 不同时也正确铺开（不要把列号写死）。
- 用户语言偏好：中文输出（CLAUDE.md §六）。Python 风格遵循 `docs/lang/lang-python.md`（类型注解、`pathlib`、`logging`、ruff 等）。

## Goals / Non-Goals

**Goals:**
- 不依赖硬编码列号，按 `N`、`M` 动态计算列偏移。
- 生成的数据严格保留 4 位小数，正态分布。
- 在「不能严格还原目标 SD」时允许 ±10% 范围；超过范围则重试，仍失败则报错而不是悄悄写入。
- 用 Levene 检验先判方差齐性（α=0.05），再选 Student t / Welch t 计算双尾 p。
- 单文件 CLI 脚本：`python scripts/generate_excel_random_data.py --input PATH [--output PATH] [--sheet NAME] [--seed INT]`。`--input` **必填**（显式传入，避免相对路径踩 CWD 与"默认文件"误改）；`--output` 不传则与 `--input` 相同，即就地修改。
- 输出可复现：支持 `--seed` 传入随机种子。

**Non-Goals:**
- 不做 GUI / Web 接口。
- 不支持非正态分布（用户明确要求正态）。
- 不处理多 Sheet 批量、跨文件合并。
- 不做样本量 N==1 的边界处理（方差未定义；目标统计量也无法成立），遇到时直接报错。
- 不支持目标均值/SD 为非数值的行：跳过并 warn。

## Decisions

### 1. 用「采样 + 线性变换」精确还原目标均值与 SD
- **方案 A（采纳）**：先用 `numpy.random.default_rng(seed).normal(0, 1, size=N)` 生成标准正态样本 `z`，然后 `x = mu_target + sd_target * (z - z.mean()) / z.std(ddof=1)`。这样得到的样本**样本均值严格等于** `mu_target`、**样本标准差（ddof=1）严格等于** `sd_target`（数值精度内）。再四舍五入到 4 位小数。
- 四舍五入后实际 SD 会**轻微偏移**目标值（量级一般 < 0.001%）。如果偏移量超过 ±10%，落入用户允许范围内（任务描述说「可以取标准差加减 10% 范围内」），仍接受；否则**最多重试 5 次不同种子**，仍失败抛 `RuntimeError`。
- **方案 B（弃用）**：纯 `rng.normal(mu, sd, N)`，靠大数定律收敛——N=10 这种小样本会有显著偏差，需要重采样很多次，浪费且不稳。
- 备注：N=2 时 `ddof=1` 仍可用；N=1 已在 Non-Goals 排除。

### 2. p 值：Levene → Student / Welch 二分支
- `scipy.stats.levene(group1, group2, center='median')`（默认即 Brown-Forsythe，对偏态更鲁棒）→ 若 `p_levene >= 0.05` 视作方差齐，调用 `scipy.stats.ttest_ind(g1, g2, equal_var=True)`；否则 `equal_var=False`（Welch）。
- 显著性阈值用 0.05（统计学惯例）。这个阈值不是用户输入参数，但代码里以常量 `LEVENE_ALPHA = 0.05` 标出，便于以后调参。

### 3. 用 openpyxl 就地写回，保留原表头与公式
- 用 `openpyxl.load_workbook(path)`（不开 `data_only`，保留公式如有；本表无公式但保险）。
- 写入前**先读取所有目标统计量再批量写入**，避免边读边改导致的状态混乱。
- 第 1 行的样本量标注列（"<count>（N）" / "<count>（M）"）：用全角括号，与原表保持一致（原表 R1 是 `'10（N）'`，AE1 是 `'12（M）'`）。
- AG-AK 表头（第 1 行）已在原表存在；脚本只写第 2 行起的统计值。如果 AG1-AK1 为空，则在第 1 行也补写表头（防御性）。

### 4. 列偏移计算（动态、零硬编码）
- 第一组数据起始列：固定 `I` 列（= 9）。这是用户在描述里直接指定的"从 I 列开始"。
- 第一组占 `[9, 9 + N - 1]`，最后一格列号 `9 + N - 1`。
- 空一列：`9 + N`。
- 第二组占 `[9 + N + 1, 9 + N + 1 + M - 1]`，最后一格列号 `9 + N + M`。
- 空一列：`9 + N + M + 1`。
- 统计列（5 列）：起始 `9 + N + M + 2`，结束 `9 + N + M + 6`。
- 当 N=10、M=12 时分别得到 9、18、20、31、33、37 — 与样本完全一致（I=9、R=18、T=20、AE=31、AG=33、AK=37）。
- 用 `openpyxl.utils.get_column_letter(idx)` 转字母，不手写映射。

### 5. 生成数据时的随机性 vs 可复现性
- 默认从 `secrets`/系统时钟取一个种子，并用 `logger.info` 打印；`--seed` 显式传入则使用之，便于复现 bug。
- 每行用 `seed + row_index` 作为子种子，保证不同指标行之间互相独立、又对外整体可复现。

### 6. 错误处理与日志
- 用 `logging` 而非 `print`（Python 准则）。`--verbose` 打印 DEBUG，否则 INFO。
- 校验失败（如 N≤1、目标 SD≤0、目标统计量缺失）→ 记录 warn 后跳过该行，不阻止其它行。
- 写入异常 → 让异常向上抛，不要 silently swallow。

## Risks / Trade-offs

- [SD 四舍五入后偏离目标 > 10%] → 重试不同种子；最多 5 次；仍失败抛错。可观测：日志里会记录每次试验的 `actual_sd / target_sd` 比值。
- [openpyxl 覆盖会丢失单元格样式] → 由于本表没有复杂格式，影响可忽略；如未来需要保格式，可改用 `openpyxl` 的 `copy_worksheet` 方案。
- [浮点 → round(4) 的微小偏差] → 用户已在描述里明确接受 ±10% SD 容忍度，且 4 位小数对偏差影响极小。
- [就地修改源文件] → 提供 `--output` 参数 + `scripts/README.md` 中明确警告"建议先备份"。
- [scipy 是较重的依赖] → 用户场景需要 Levene + t 检验，自己实现 Welch / Levene 反而易错；接受这个体积成本。
